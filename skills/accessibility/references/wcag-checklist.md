# WCAG 2.1 Quick Reference Checklist

## Level A (Minimum)

### Perceivable
- [ ] **1.1.1** All images have meaningful alt text (or empty alt="" for decorative)
- [ ] **1.2.1** Audio/video has text alternatives (captions or transcripts)
- [ ] **1.3.1** Information is conveyed through structure (headings, lists, tables), not just visually
- [ ] **1.3.2** Reading order makes sense when CSS is disabled
- [ ] **1.4.1** Color is not the only way information is conveyed

### Operable
- [ ] **2.1.1** All functionality works with keyboard alone (no mouse required)
- [ ] **2.1.2** No keyboard traps — user can always tab away
- [ ] **2.4.1** Skip navigation link available
- [ ] **2.4.2** Pages have descriptive titles
- [ ] **2.4.3** Focus order is logical (left-right, top-bottom)

### Understandable
- [ ] **3.1.1** Page language is specified (`<html lang="en">`)
- [ ] **3.2.1** Focus doesn't trigger unexpected changes
- [ ] **3.3.1** Input errors are clearly identified and described
- [ ] **3.3.2** Labels or instructions for user inputs

### Robust
- [ ] **4.1.1** No duplicate IDs, valid HTML structure
- [ ] **4.1.2** All UI components have accessible name and role

## Level AA (Recommended)

### Perceivable
- [ ] **1.4.3** Text contrast ratio ≥ 4.5:1 (3:1 for large text)
- [ ] **1.4.4** Text can be resized to 200% without loss of content
- [ ] **1.4.5** Text is used instead of images of text
- [ ] **1.4.10** Content reflows at 320px width (no horizontal scroll)
- [ ] **1.4.11** UI component contrast ratio ≥ 3:1

### Operable
- [ ] **2.4.6** Headings and labels are descriptive
- [ ] **2.4.7** Focus indicator is visible
- [ ] **2.5.3** Accessible name matches visible label

### Understandable
- [ ] **3.2.3** Navigation is consistent across pages
- [ ] **3.2.4** Components with same function identified consistently
- [ ] **3.3.3** Error suggestions provided when possible
- [ ] **3.3.4** Error prevention for legal/financial submissions
